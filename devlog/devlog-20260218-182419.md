# Daily Doom Development Log - 2026-02-18

## Issue #17: Enemy Damage System - Demons Attack Player

Implemented the core enemy-to-player combat mechanic. Enemies now actively deal damage when in attack range with line of sight.

**Changes:**
- Enemies deal type-specific damage (Imp: 10, Guard: 15, Soldier: 20, Demon: 35)
- Attack cooldowns per enemy type (1.2s - 3s depending on type)
- Player invincibility frames (500ms) prevent spam damage
- Red screen flash on damage via HUD system
- Pain sound effect (procedural sawtooth sweep) on hit
- Player death and respawn system (2s respawn delay)
- Game pauses updates while player is dead
- Player death sound effect (layered frequency sweep)
- Added T2-08 playtester test validating the complete damage pipeline

**Files modified:** enemy-behaviors.js, enemy.js, player.js, sound-engine.js, game.js, tests.js

## Issue #18: Dynamic Enemy Movement & Pathfinding

Implemented A* pathfinding so enemies can navigate around walls and obstacles to reach the player.

**Changes:**
- A* pathfinding algorithm added to GameMap with octile distance heuristic
- Diagonal movement with corner-cutting prevention
- Enemies use pathfinding when they lose line of sight to the player
- Path recalculation every 500ms for performance
- Stuck detection: enemies that haven't moved for 30+ frames trigger path recalculation
- Both enhanced AI and basic AI use pathfinding
- Added T2-09 playtester test validating pathfinding and varied enemy speeds

**Files modified:** map.js, enemy-behaviors.js, enemy.js, tests.js
